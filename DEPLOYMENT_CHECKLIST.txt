## 🚀 QUICK START (3 STEPS)

### Step 1: Setup Environment
```bash
cp backend/.env.production.example backend/.env
nano backend/.env  # Fill in required values
```

### Step 2: Build
```bash
docker-compose build --no-cache
```

### Step 3: Deploy
```bash
docker-compose up -d
curl http://localhost:5000/api/health  # Should return 200
```

## 🎓 WHAT YOU SHOULD KNOW

**Passwords & Secrets**
- JWT_SECRET must be 32+ characters, random, and unique per environment
- Database passwords should be 16+ characters
- Store all secrets in .env (never in code)
- .env file is in .gitignore (never commit to git)

**Environment Variables**
- Required: PORT, NODE_ENV, MONGODB_URI, JWT_SECRET, CLIENT_URL
- Optional: Stripe, Razorpay, Cloudinary, OpenAI, SMTP
- Missing required vars = app won't start
- Invalid vars = app exits with validation errors

**Database**
- Use MongoDB Atlas for production (not self-hosted)
- Enable backups before going live
- Encrypt connections (TLS)
- Restrict network access

**Deployment**
- Docker Compose for simplified deployment
- GitHub Actions for automated tests & deployment
- Nginx for reverse proxy and SSL/TLS
- Rate limiting to prevent abuse
